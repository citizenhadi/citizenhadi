### II. Component-Based Development (React JS)

Focus on building scalable Single Page Applications (SPAs) and mastering the React ecosystem.

#### Core React Concepts
* **JSX (JavaScript XML):** Writing declarative UI elements and embedding JavaScript within HTML structure.
* **Components:** Mastering Functional Components and understanding the differences with Class Components (HOCs, lifecycle methods).
* **Props:** Passing data and functions between components for communication (Parent-to-Child).
* **State Management (Local):** Utilizing the `useState` hook for managing local component data and triggering re-renders.
* **Component Lifecycle & Side Effects:** Using the `useEffect` hook for tasks like data fetching, subscriptions, and manual DOM manipulation.
* **Conditional & List Rendering:** Displaying elements based on conditions and efficiently rendering dynamic lists using the `key` prop.

#### Advanced React Features (Hooks)
* **Context API & `useContext`:** Managing and sharing global state data without prop drilling.
* **`useReducer`:** Handling complex state logic that involves multiple sub-values or next state based on the previous one.
* **`useRef`:** Accessing the underlying DOM elements and maintaining mutable values across renders.
* **Custom Hooks:** Creating reusable logic functions to share non-visual behavior between components.

#### Styling & Interactivity
* **Styling Approaches:** Implementing styling using pure CSS, CSS Modules, CSS-in-JS libraries, and integrating frameworks like Tailwind CSS or Bootstrap in React projects.
* **Event Handling:** Capturing and responding to user interactions with synthetic events.
* **Forms & Controlled Components:** Building and validating forms by controlling form input values through React state.

#### Routing & Data Flow
* **React Router (SPA):** Implementing client-side routing, navigation, nested routes, and route parameters for Single Page Applications.
* **Asynchronous Operations:** Handling API requests using `Fetch` or `Axios` within the `useEffect` hook.
* **State Management (Global):** Introduction to major patterns or libraries like **Redux/Redux Toolkit** (Actions, Reducers, Store, Dispatch) for enterprise-level applications.

#### Tooling & Ecosystem
* **Module Bundlers:** Working with **Vite** or **Webpack** for project setup and optimization.
* **NPM/Yarn:** Managing project dependencies and scripts.
* **Debugging:** Using React Developer Tools browser extension for component inspection, state, and prop analysis.
* **Performance Optimization:** Concepts like memoization using `React.memo` and `useCallback`/`useMemo` to prevent unnecessary re-renders.

### III. Full-Stack Rendering & Performance (Next.js)

Expertise in building high-performance, SEO-friendly, and production-ready full-stack applications with React.

#### Core Framework Concepts
* **Next.js Fundamentals:** Understanding the App Router vs. Pages Router architectures and project structure.
* **Routing:** Implementing simple and complex routing, nested routes, dynamic routes, and route groups.
* **Layouts & Templates:** Defining shared UI structure across multiple routes for consistent navigation.
* **Metadata & SEO:** Managing application titles, descriptions, and Open Graph tags for Search Engine Optimization (SEO).

#### Rendering Strategies & Performance
* **Server-Side Rendering (SSR):** Fetching data on the server for improved performance and SEO.
* **Static Site Generation (SSG):** Pre-rendering pages at build time for ultimate speed and efficiency.
* **Incremental Static Regeneration (ISR):** Updating static content after deployment without rebuilding the entire site.
* **Client-Side Rendering (CSR):** When and how to fetch data directly in the browser (using React's useEffect).

#### Data Fetching & Full-Stack Capabilities
* **Server Components (React 18+):** Utilizing Server Components for zero-bundle-size, fast loading UIs.
* **Server Actions:** Implementing direct server mutations and form handling without dedicated API routes (Full-Stack functionality).
* **API Routes (Back-End):** Creating robust serverless functions within the Next.js project to handle API endpoints.
* **Database Integration:** Connecting and querying databases (e.g., MongoDB, PostgreSQL) directly from Server Components or API Routes.

#### Middleware & Deployment
* **Middleware:** Writing custom logic that executes before a request is completed (e.g., authentication, redirection).
* **Environment Variables:** Securely managing application configurations and secrets.
* **Deployment:** Deploying Next.js applications to platforms like Vercel, Netlify, or custom servers.
* **Image Optimization:** Utilizing the Next.js `Image` component for automatic size optimization and lazy loading.
